# custom-nginx

## Nginxコンテナを起動する

Dockerで作成した `nginx-custom:1.0.0` イメージを使って、Nginxコンテナを起動する。

### 実行するコマンド

```bash
docker run --rm --detach --publish 8080:80 --name web nginx-custom:1.0.0
```

### コマンド全体の意味

`nginx-custom:1.0.0` イメージから `web` という名前のコンテナを起動する。
ホスト側の8080番ポートとコンテナ側の80番ポートを紐づけることで、ブラウザから `http://localhost:8080` にアクセスできるようにする。

### オプションの説明

* `--rm`

  コンテナの停止時に、コンテナを自動的に削除する。

* `--detach`

  コンテナをバックグラウンドで実行する。
  これにより、ターミナルを占有せずにコンテナを起動できる。

* `--publish 8080:80`

  ホスト側の8080番ポートと、コンテナ側の80番ポートを紐づける。
  これにより、ブラウザで `http://localhost:8080` にアクセスすると、コンテナ内のNginxにアクセスできる。

* `--name web`

  コンテナに `web` という名前を付ける。
  コンテナ名を付けておくと、停止やログ確認などの操作がしやすくなる。

* `nginx-custom:1.0.0`

  起動するDockerイメージを指定する。
  この例では、`nginx-custom` というイメージの `1.0.0` タグを使用している。

### 起動確認

```bash
docker ps
```

起動中のコンテナ一覧を確認する。
`web` という名前のコンテナが表示されていれば、起動できている。

### ブラウザで確認

```text
http://localhost:8080
```

ホスト側の8080番ポートにアクセスすることで、コンテナ内で起動しているNginxにアクセスできる。

### コンテナの停止

```bash
docker stop web
```

`web` という名前のコンテナを停止する。
今回のコマンドでは `--rm` を指定しているため、停止するとコンテナは自動的に削除される。
